# Firmware Engineering Course Tier List

## S Tier — Must Take
- **Computer Architecture:** designing a 5-stage CPU (e.g., RISC-V), understanding how processors work, assembly exposure; foundational for reading datasheets and writing hardware-aware code
- **Data Structures & Algorithms:** essential for interviews (DFS, BFS, hashmaps, arrays); teaches you how to think through problems; C++ and Python fluency expected
- **Embedded Systems:** programming microcontrollers, low-level C, peripherals, bitwise operations, pointer arithmetic, hands-on hardware (Arduino/RPi)

## A Tier — Strongly Recommended
- **Electrical Circuit Analysis:** foundational for talking to EEs, understanding board designs, differentiating yourself from pure CS candidates; not S-tier only because entry-level jobs don't always require deep circuit knowledge
- **Machine Learning:** TinyML, edge deployment, model quantization/compression is a huge trend; defense and autonomous systems demand it; deploy models on microcontrollers for local inference
- **Wireless Communication Systems:** most embedded products have antennas (Bluetooth, RF, Wi-Fi); understand frequency bands, impedance matching, antenna design; pairs with DSP

## B Tier — Good to Know
- **Real-Time Operating Systems (RTOS):** most professional firmware uses an RTOS; learnable on your own with an ESP32 + FreeRTOS; may not need a full college course
- **Digital Logic Design:** SystemVerilog/VHDL, logic gates, FPGA basics; prerequisite for computer architecture; good for understanding how processors are built
- **Operating Systems:** more relevant for higher-level firmware roles (kernel dev, FPGA low-latency systems); not frequently asked in embedded interviews
- **Object-Oriented Programming:** necessary stepping stone to DSA; everyone writing code should know OOP

## C Tier — Nice to Have
- **Assembly Language Programming:** important to understand but picked up naturally through computer architecture and embedded systems courses; not worth a standalone 3-month course
- **Linear Algebra:** foundation for ML; useful for matrix operations; becoming relevant with TinyML roles

## D Tier — Situational
- **Networking:** rarely needed beyond what you can Google; not a blocker for most firmware roles
- **Control Systems Engineering:** only if going into robotics specifically

## General Advice
- Firmware engineering is **extremely broad** — find a sub-field you're passionate about and specialize
- Communication protocols (SPI, CAN, I2C) are usually covered in Embedded Systems 1 & 2
- **Side projects** are essential alongside coursework
